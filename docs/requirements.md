# 要件定義

## 目的

このプロジェクトは、LINE BotおよびMessaging APIの基本的な使い方を個人的に学習するためのもの。

本番リリースや外部ユーザーへの提供は目的とせず、Webhook、イベント処理、応答メッセージ、画像返信、スタンプ返信などの基本機能を実際に動かしながら理解することを目的とする。

## 対象ユーザー

- 開発者本人
- LINE Botの基本的な仕組みを学習したい個人利用者

## ユースケース

- 開発者がLINEでbotにメッセージを送信する。
- botが受信したメッセージ内容に応じて返信する。
- テキスト、画像、スタンプなど複数種類の返信を試す。
- 想定していないメッセージに対しては、おうむ返しなどの簡単な応答を返す。

## 機能要件

- LINEから送信されるWebhookイベントを受け取れること。
- Webhookイベントの署名検証を行えること。
- ユーザーからのテキストメッセージを判定できること。
- 特定のメッセージに対してテキスト返信できること。
- 特定のメッセージに対して画像返信できること。
- 特定のメッセージに対してスタンプ返信できること。
- 未定義のメッセージに対して、おうむ返しなどの基本応答を返せること。

## 非機能要件

- 原則として無料で利用できる範囲に収める。
- LINE公式アカウントは無料プランの範囲で利用する。
- Messaging APIは無料枠内で利用し、応答メッセージを中心に実装する。
- 有料サービス、有料プラン、有料APIは利用しない。
- 個人学習用のため、大規模な性能要件や高可用性は求めない。
- チャネルシークレットやアクセストークンなどの機密情報は環境変数で管理する。
- まずはデータベースを使わず、必要になった場合のみSQLiteなど無料で扱える選択肢を検討する。

## 対象外

- 本番リリース
- 外部ユーザーへの公開提供
- 有料プランの利用
- 課金が発生する追加メッセージ送信
- 管理画面
- 認証機能
- 複雑なデータ永続化
- 大規模配信、ブロードキャスト、マーケティング用途
